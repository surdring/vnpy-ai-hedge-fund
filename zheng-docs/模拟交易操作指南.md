## 3. LLM 提供商配置

AI Agent 需要 LLM 来生成交易信号。支持两种方式：

### 3.1 方式一：Ollama 本地模型（推荐，免费）

**步骤 1**：安装 Ollama

从 [https://ollama.com/download](https://ollama.com/download) 下载并安装。

**步骤 2**：下载模型

```powershell
ollama pull llama3.2       # 轻量模型，约 2GB
# 或
ollama pull qwen2.5:7b     # 通义千问，中文能力更强
# 或
ollama pull deepseek-r1:8b # DeepSeek 推理模型
```

**步骤 3**：验证模型可用

```powershell
ollama list
# 输出示例：
# NAME              ID              SIZE      MODIFIED
# llama3.2:latest   a80c4f17acd5    2.0 GB    2 days ago
```

**步骤 4**：配置 `config/default_settings.json`

```json
{
  "llm_provider": "Ollama",
  "llm_model_name": "llama3.2"
}
```

## 4. 数据准备

模拟交易需要历史 K 线数据。AI Agent 的分析依赖价格历史，最低要求每个标的 **20 根日线 Bar**。

### 4.1 方式一：通过 vnpy GUI 下载（推荐）

1. 启动 vnpy GUI（见第 6 节）
2. 点击菜单 **功能 → 数据管理**
3. 在数据管理界面中：
   - 选择数据服务（如 RQData）
   - 输入本地代码（如 `IF88.CFFEX` 对应沪深300股指期货主力连续）
   - 选择 K 线周期：`d`（日线）
   - 选择起止日期
   - 点击 **下载数据**

### 4.2 方式二：通过代码下载

```python
# download_data.py
from vnpy.trader.constant import Interval
from vnpy.trader.database import get_database
from vnpy.trader.datafeed import get_datafeed
from vnpy.trader.object import HistoryRequest

def download(symbol: str, exchange: str, start: str, end: str):
    """下载历史数据到本地数据库"""
    database = get_database()
    datafeed = get_datafeed()

    req = HistoryRequest(
        symbol=symbol,
        exchange=exchange,
        start=datetime.strptime(start, "%Y%m%d"),
        end=datetime.strptime(end, "%Y%m%d"),
        interval=Interval.DAILY,
    )

    bars = datafeed.query_bar_history(req)
    if bars:
        database.save_bar_data(bars)
        print(f"下载完成: {symbol}.{exchange}, 共 {len(bars)} 条")
    else:
        print(f"无数据: {symbol}.{exchange}")

if __name__ == "__main__":
    # 示例：下载沪深300股指期货主力连续
    download("IF88", "CFFEX", "20240101", "20260620")
```

### 4.3 支持的标的格式

| 品种 | 代码示例 | 交易所 | 备注 |
|------|---------|--------|------|
| 沪深300股指期货 | `IF88.CFFEX` | CFFEX | 主力连续 |
| 螺纹钢期货 | `RB88.SHFE` | SHFE | 主力连续 |
| 铁矿石期货 | `I88.DCE` | DCE | 主力连续 |
| 原油期货 | `SC88.INE` | INE | 主力连续 |
| 黄金期货 | `AU88.SHFE` | SHFE | 主力连续 |
| 上证50ETF期权 | `510050.SSE` | SSE | 股票 |
| 创业板ETF | `159915.SZSE` | SZSE | 股票 |

> **注意**：AI Agent 的 19 个分析师 Agent 中的部分（如估值、基本面、新闻情绪）依赖美股数据源。对于国内期货/股票，建议只选择 `technical_analyst`、`sentiment_analyst`、`growth_analyst` 等不依赖美股财务数据的 Agent。

## 5. 配置模拟交易参数

### 5.1 核心配置文件

编辑 `config/default_settings.json`：

```json
{
  "enabled": true,
  "enable_auto_trading": false,
  "trigger_frequency": "daily",
  "selected_analysts": [
    "technical_analyst",
    "valuation_analyst",
    "sentiment_analyst",
    "growth_analyst"
  ],
  "llm_model_name": "llama3.2",
  "llm_provider": "Ollama",
  "max_position_ratio": 0.2,
  "signal_cooldown": 300,
  "fallback_strategy": "sma_cross",
  "rpc_host": "127.0.0.1",
  "rpc_server_port": 9001,
  "rpc_agent_port": 9002,
  "gateway_name": "PAPER",
  "order_type": "LIMIT"
}
```

### 5.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | false | **设为 true** 启用 AI Agent |
| `enable_auto_trading` | bool | false | **模拟交易建议保持 false**，手动确认信号后再下单。设为 true 则自动下单 |
| `trigger_frequency` | str | "daily" | 触发频率：`daily`（日频）、`tick`、`bar` |
| `selected_analysts` | list | [] | 选用的分析师 Agent，空列表 = 全部 19 个 |
| `llm_model_name` | str | "llama3" | LLM 模型名称 |
| `llm_provider` | str | "Ollama" | LLM 提供商 |
| `max_position_ratio` | float | 0.2 | 单标的最大仓位比例（占总资金） |
| `signal_cooldown` | int | 300 | 信号冷却时间（秒），避免频繁触发 |
| `fallback_strategy` | str | "hold" | LLM 不可用时的降级策略：`hold`（持仓不动）、`sma_cross`（SMA 交叉） |
| `gateway_name` | str | "" | 交易接口名称，模拟交易填 `"PAPER"` |
| `order_type` | str | "LIMIT" | 订单类型：`LIMIT`（限价）、`MARKET`（市价） |

## 7. 启动模拟交易（CLI 方式）

### 7.1 纯脚本模式（无 GUI）

```python
# paper_trading_cli.py
"""CLI 模式模拟交易脚本"""
from vnpy.event import EventEngine
from vnpy.trader.engine import MainEngine
from vnpy.trader.constant import Interval
from vnpy.trader.object import HistoryRequest
from datetime import datetime

from vnpy_ai.app import AiHedgeFundApp
from vnpy_ai.config import AiSettings


def main():
    # 1. 初始化事件引擎和主引擎
    event_engine = EventEngine()
    event_engine.start()
    main_engine = MainEngine(event_engine)

    # 2. 加载 AI Agent 策略
    ai_engine = main_engine.add_app(AiHedgeFundApp)

    # 3. 配置 AI Agent（模拟交易，不自动下单）
    ai_settings = AiSettings(
        enabled=True,
        enable_auto_trading=False,          # 模拟交易先手动确认
        trigger_frequency="daily",
        selected_analysts=[
            "technical_analyst",
            "sentiment_analyst",
            "growth_analyst",
        ],
        llm_model_name="llama3.2",
        llm_provider="Ollama",
        max_position_ratio=0.2,
        signal_cooldown=300,
        fallback_strategy="sma_cross",
        gateway_name="PAPER",
        order_type="LIMIT",
    )
    ai_engine.set_config(ai_settings)

    # 4. 运行一次 AI 决策
    tickers = ["IF88.CFFEX", "RB88.SHFE"]  # 分析标的
    result = ai_engine.run_workflow(
        tickers=tickers,
        start_date="2026-03-01",
        end_date="2026-06-21",
    )

    # 5. 查看结果
    print(f"工作流状态: {'降级' if result.degraded else '正常'}")
    for ticker, decision in result.decisions.items():
        print(f"  {ticker}: {decision.action} | 置信度: {decision.confidence}% | 理由: {decision.reasoning[:100]}")

    # 6. 查看各 Agent 信号
    for ticker, signals in result.analyst_signals.items():
        print(f"\n{ticker} 各 Agent 信号:")
        for agent_name, signal_data in signals.items():
            print(f"  {agent_name}: {signal_data.get('signal', 'N/A')}")

    main_engine.close()


if __name__ == "__main__":
    main()
```

运行：
```powershell
python paper_trading_cli.py
```

### 7.2 定时运行模式（日频调度）

```python
# scheduled_paper_trading.py
"""定时调度模拟交易"""
import time
import schedule  # pip install schedule
from datetime import datetime

def run_ai_cycle():
    """每日 AI 决策循环"""
    print(f"[{datetime.now()}] 开始 AI 决策...")
    # ... 调用 run_workflow() 逻辑 ...
    print(f"[{datetime.now()}] 决策完成")

# 每个交易日 09:30 运行
schedule.every().day.at("09:30").do(run_ai_cycle)

while True:
    schedule.run_pending()
    time.sleep(60)
```

```
                                    │       ▼          ▼       │
                                    │  Risk Manager  Portfolio │
                                    │       │          │       │
                                    │       ▼          ▼       │
                                    │  信号监控面板 ← 决策     │
                                    │       │                  │
                                    │       ▼                  │
                                    │  手动/自动下单           │
                                    └─────────────────────────┘
```

### 9.2 AI 决策流程详解

1. **行情触发**：当 tick/bar 行情到达时（或定时触发），AiHedgeFundEngine 被唤醒
2. **数据获取**：DataAdapter 从 vnpy 数据库/行情中获取价格数据
3. **Agent 分析**：选定的分析师 Agent 并行调用 LLM，生成各自的信号（bullish/bearish/neutral）
4. **风险控制**：Risk Manager 计算波动率、VaR、相关性，设定仓位上限
5. **投资组合决策**：Portfolio Manager 综合所有信号和风控限制，生成最终决策（buy/sell/short/cover/hold）
6. **订单执行**：如开启自动交易，OrderAdapter 将决策转换为 OrderRequest 并发送到 PAPER 接口
7. **状态同步**：同步持仓、订单状态到工作流

### 9.3 降级策略

当 LLM 不可用时（无网络、API Key 无效等），系统自动降级：

| 降级策略 | 行为 |
|---------|------|
| `hold` | 所有标的不做任何操作，持仓不动 |
| `sma_cross` | 使用 SMA(5) 和 SMA(20) 交叉信号，金叉买入，死叉卖出 |

## 11. 常见问题排查

### 11.1 LLM 连接失败

**症状**：工作流返回 `degraded: true`，所有决策为 `hold`

**排查**：
```powershell
# 测试 Ollama 连接
ollama list

# 测试 API 连接
curl http://localhost:11434/api/tags

# 检查 .env 文件
type .env
```

**解决**：
- Ollama 未启动：运行 `ollama serve`
- API Key 未配置：检查 `.env` 文件
- 网络问题：检查代理设置

### 11.2 数据为空

**症状**：`get_prices()` 返回空列表，Agent 无法分析

**排查**：
```python
from vnpy.trader.database import get_database
database = get_database()
bars = database.load_bar_data("IF88", "CFFEX", Interval.DAILY, start, end)
print(f"数据条数: {len(bars)}")
```

**解决**：
- 确认已下载历史数据（见第 4 节）
- 检查合约代码格式：`symbol.EXCHANGE`
- 确认数据服务已配置

### 11.3 模拟交易接口未连接

**症状**：下单时报错 `gateway_name is required`

**解决**：
- 确保 `config/default_settings.json` 中 `gateway_name` 设为 `"PAPER"`
- 在 GUI 中手动连接 PAPER 接口
- 确认已安装 `vnpy_paperaccount`：`pip install vnpy_paperaccount`

### 11.4 所有 Agent 信号都是 neutral

**症状**：Agent 输出全为 HOLD

**排查**：
- 检查 `selected_analysts` 是否配置正确
- 查看降级策略是否触发
- 检查 LLM 是否正常响应

```python
# 手动测试一个 Agent
from vnpy_ai.agents.technical_analyst import technical_analyst_agent
from vnpy_ai.workflow.runner import create_initial_state
from vnpy_ai.data_adapter import DataAdapter

state = create_initial_state(["IF88.CFFEX"], {}, model_name="llama3.2", model_provider="Ollama")
result = technical_analyst_agent(state, "technical_analyst")
print(result)
```

### 11.5 降级策略触发

**症状**：`degraded: true` 且使用 SMA 交叉信号

**原因**：LangGraph 工作流未成功执行（导入失败或执行异常）

**这不是错误**：这是设计的容错机制。降级模式下系统仍可运行，但信号质量可能不如 LLM 模式。

**如果想强制使用 LLM 模式**：
```powershell
pip install langchain langchain-core langgraph langchain-ollama
```
