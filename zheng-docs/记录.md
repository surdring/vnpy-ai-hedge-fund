VeighNa + AI Hedge Fund 全量整合实施计划
Summary
按 Phase 1-3 全量交付：新增 vnpy_ai/ 插件包、vnpy_ai_web/ Web 编辑器、根级项目配置、集成测试、示例、配置与部署文件。保持 ai-hedge-fund/ 只读；vnpy/ 仅新增方案规定的 AI 事件常量和数据类。AI 默认关闭，自动交易默认关闭，LLM 不可用时降级为 hold/信号模式。
Key Changes
建立根级 pyproject.toml，使用 Python 3.11+，声明 vnpy_ai 为主包；LangChain、FastAPI、Web 相关依赖放入可选 extras 或独立进程依赖，避免启动 vnpy 时默认加载。
在 vnpy_ai/ 实现核心接口：AiHedgeFundApp、AiHedgeFundEngine、DataAdapter、OrderAdapter、EventAdapter、RpcBridge、config、workflow、agents、llm、monitoring。
从 ai-hedge-fund/src 复制 Agent、graph、LLM、data models、progress 等必要代码到 vnpy_ai/agents、vnpy_ai/workflow、vnpy_ai/llm，保留原始作者/许可信息，并把 src.tools.api 数据入口改为调用 vnpy_ai.data_adapter。
在 vnpy/trader/event.py 新增 EVENT_AI_SIGNAL、EVENT_AI_DECISION、EVENT_AI_DECISION_ORDER、EVENT_AI_ERROR、EVENT_AI_STATUS；在 vnpy/trader/object.py 新增 AiSignalData、AiDecisionData dataclass。
实现 RPC 双进程通信：vnpy 主进程提供市场数据、组合状态、下单、撤单、订阅接口；Agent 进程接收行情推送、订单状态、心跳；默认 localhost 9001/9002，可通过配置覆盖。
实现数据适配：vnpy 本地数据库 load_bar_data 转 AI HF Price；OmsEngine tick/position/account 转 portfolio；datafeed 扩展接口保留为财务指标、新闻、内部交易的可选来源；缺失数据返回空列表并触发降级。
实现订单适配：buy/sell/short/cover/hold 转 OrderRequest，校验合约、方向、价格、数量、仓位与资金；enable_auto_trading=False 时只发 AI 决策事件，不直接下单。
迁移 Web：复制 ai-hedge-fund/app 到 vnpy_ai_web/，后端 import 改为 vnpy_ai.*，保留 React Flow、SSE 进度、API Key 管理；读取 API Key 时前端只返回脱敏值。
增加配置与示例：config/default_settings.json、agent profiles、LLM providers、.env.example、examples/ai_agent_strategy、examples/multi_agent_backtest。
增加生产化内容：统一回测入口、结构化日志、审计日志、进程存活检测、Dockerfile.core/agent/web、docker-compose、部署文档。
Public Interfaces
Python App：vnpy_ai.app.AiHedgeFundApp 可通过 MainEngine.add_app(AiHedgeFundApp) 加载。
Engine API：start_agent_process()、stop_agent_process()、run_workflow()、submit_decision()、sync_portfolio()、set_config()、get_status()。
Data API：get_prices(ticker, start_date, end_date)、get_financial_metrics(...)、get_company_news(...)、get_portfolio()，返回 AI HF 兼容 Pydantic 模型。
RPC 消息：统一 {type, timestamp, request_id, payload}，覆盖 market data、portfolio、submit/cancel order、agent signal、order status、heartbeat。
Web API：保留 /hedge-fund/run、/hedge-fund/backtest、/hedge-fund/agents、/language-models、/api-keys、/flows，实现路径改接 vnpy_ai。
Test Plan
单元测试：数据转换、订单映射、事件转换、RPC 消息编解码、API Key 脱敏、LLM 后处理、fallback 策略。
集成测试：本地 SQLite bar 数据跑单次 Agent 分析；mock RPC 回环；vnpy MainEngine.add_app 加载；事件总线收到 AI signal/decision/error/status。
端到端测试：禁用自动交易时只产生决策事件；启用 mock gateway 时订单正确发送；Web /run SSE 输出 start/progress/complete；回测结果与独立 AI HF 同输入结果一致。
前端验证：npm run build、关键页面 Playwright 冒烟测试、API Key 不明文展示。
最终命令：pytest tests/ -v、ruff check vnpy_ai/、mypy vnpy_ai/、npm run build in vnpy_ai_web/frontend、docker compose config。
Assumptions
本次按用户确认的 Phase 1-3 全量范围规划。
不修改 ai-hedge-fund/ 源文件；vnpy/ 只做事件常量和 AI dataclass 的最小新增。
自动交易、AI Agent 启动、云端 LLM 均默认关闭；本地 Ollama 优先，BYOK 可选。
生产 7 天稳定运行验收作为部署后 soak test，不阻塞本地代码合并，但必须在任务状态中标记为待生产验收。
